# 1. Record architecture decisions

Date: 2026-04-10

## Status

Accepted

## Context

We need to record the architectural decisions made on this project so that future contributors and maintainers understand the reasoning behind major technical choices (like using Astro, Cloudflare D1, and Dynamic Worker Loaders).

## Decision

We will use Architecture Decision Records (ADRs). This repository will store them in the `docs/adr` directory. Each new major architecture shift will be documented as a new Markdown file in this structure.

## Consequences

* See [Architecture Decision Records](https://adr.github.io/) for more context.
* It will be easier to onboard new developers since the historical context of technical trade-offs is preserved.
